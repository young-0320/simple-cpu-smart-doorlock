0. 프로젝트 구조 간단 소개

설명 멘트는 짧게 진행
우리가 만든 스마트 도어의 기능만 간략하게 설명 (코드와 관련된 자세한 내용은 실제 발표에서 진행)

1. 초기 상태 확인
1) 보드 리셋
2) 도어락이 닫힘 상태인지 확인
3) 아무런 LED가 켜져 있지 않으면 닫혀있는 상태


2. 초기 비밀번호 입력으로 도어락 열림 확인
1) 초기 비밀번호 입력
예: 1 → 2 → 3 → 4
2) Unlock LED 또는 Open LED 점등 확인
3) 문을 닫으면 닫힘상태가 되듯이 reset버튼을 누르면 켜져있던 unlock LED가 꺼짐


3. 입력 흐름 중 비밀번호 포함 여부 확인
1) 비밀번호가 1234라고 가정
2) 전체 입력을 5 → 6 → 1 → 2 → 3 → 4 → 7 → 8처럼 입력
3) 중간에 1234가 연속으로 포함되면 도어락 열림 확인


4. 입력 취소 기능 확인
1) 일부 숫자 입력
예: 1 → 2 → 9
2) 취소 버튼 누르기 --> 마지막 입력 9가 삭제된 상태로 진행
3) 이어서 3 → 4 입력 --> 결과적으로 1 → 2 → 3 → 4가 되어 도어락 열림 확인


5. 비밀번호 오류 1회 확인
1) 틀린 비밀번호 입력
예: 1 → 1 → 1 → 1
2) Fail LED 또는 Red LED 1개 점등 확인


6. 비밀번호 오류 2회 확인
1) 다시 틀린 비밀번호 입력
예: 2 → 2 → 2 → 2
2) 다른 LED 위치에 Red LED 상태 확인


7. 비밀번호 오류 3회 및 입력 지연 확인
1) 세 번째로 틀린 비밀번호 입력
예: 3 → 3 → 3 → 3
2) 외부 Red LED 1개 점등 확인
3) 입력 제한 상태 진입 확인
4) 10초 동안 입력해도 동작하지 않는지 확인
5) 10초 후 다시 입력 가능해지는지 확인


8. 마스터키 입력으로 즉시 열림 확인
1) 마스터키 입력
예: 실제 BRAM에 저장한 마스터키
2) 도어락 즉시 열림 확인


9. 비밀번호 변경 모드 진입
1) 비밀번호 변경 버튼 또는 change mode 입력
2) 새 비밀번호 입력 (8자리로)
3) 동일한 비밀번호를 한 번 더 입력 (8자리로)
4) 두 입력이 일치하면 비밀번호 변경 완료 LED 또는 상태 확인 (이건 추후에 가능하면)


10. 변경된 비밀번호로 도어락 열림 확인
1) 기존 비밀번호 입력
예: 1 → 2 → 3 → 4
2) 열리지 않는지 확인
3) 새 비밀번호 입력
예: 2 → 5 → 8 → 0
4) 새 비밀번호가
4) 도어락 열림 확인
