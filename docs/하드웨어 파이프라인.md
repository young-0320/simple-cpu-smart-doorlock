# 하드웨어 파이프라인

## 1. 전체 시스템 구조

```
[물리 입력]
  pmod_key[9:0]  ─┐
  btn_input       │
  btn_confirm     ├─► input_handler ─► in_port[8:0]   ─┐
  btn_cancel      │                                    │
  btn_change      │                                    │
  btn_master      ┘                                    │
                                                       ▼
                                                  ┌─────────┐
                              bram_rdata[31:0] ◄──┤         ├──► bram_addr[11:0]
                                                  │ top_cpu │
                              (BRAM: doorlock.coe)│         ├──► bram_wdata[31:0]
                              ◄──────────────────►│         │
                                                  │         ├──► bram_we
                                                  └────┬────┘
                                                       │ out_port[3:0]
                                                       ▼
                                                 output_handler
                                                  │         │
                                                  ▼         ▼
                                              door_open   led[3:0]
```

클럭: 보드 125MHz → `clk_wiz_0` → 7MHz (`clk_cpu`) → 전 모듈 공급  
리셋: `ext_reset | ~clk_locked` = `sys_reset` (active-high, 동기)

## 2. input_handler

**역할**: 물리 입력을 CPU가 읽을 수 있는 `in_port[8:0]`로 변환

### 입력

| 신호 | 폭 | 핀 | 설명 |
|---|---|---|---|
| `pmod_key` | 10 | JE/JD | PMOD one-hot 숫자 키패드 |
| `btn_input` | 1 | BTN1 P16 | 자리 입력 확정 버튼 |
| `btn_confirm` | 1 | BTN2 K19 | 전체 비밀번호 확정 버튼 |
| `btn_cancel` | 1 | BTN3 Y16 | 취소 버튼 |
| `btn_change` | 1 | SW0 G15 | 비밀번호 변경 모드 스위치 |
| `btn_master` | 1 | SW1 P15 | 마스터키 스위치 |

### 처리 단계

```
pmod_key[9:0]
    │
    ▼ (×10) debouncer
    │  2-Stage Synchronizer → 20ms 안정화 카운터
    ▼
key_level[9:0]  (디바운싱된 level)
    │
    ▼ casez 우선순위 인코더 (낮은 번호 우선)
    │  10-bit one-hot → 4-bit binary
    ▼
key_num[3:0]  ──────────────────────────────────► in_port[3:0]

btn_input / confirm / cancel / change / master
    │
    ▼ (×5) debouncer
    │  → btn_pulse (상승 에지 1클럭 펄스)
    ▼
self-clearing latch
    │  pulse → SET(1), 다음 클럭 CLR(0)
    │  pulse와 CLR 동시 → SET 우선
    ▼
latch_input   ──────────────────────────────────► in_port[4]
latch_confirm ──────────────────────────────────► in_port[5]
latch_cancel  ──────────────────────────────────► in_port[6]
latch_change  ──────────────────────────────────► in_port[7]
latch_master  ──────────────────────────────────► in_port[8]
```

### 출력

| 비트 | 의미 | 유지 기간 |
|---|---|---|
| `in_port[3:0]` | 현재 눌린 키 숫자 (0~9) | pmod 눌려있는 동안 |
| `in_port[4]` | 자리 입력 확정 | 1 클럭 |
| `in_port[5]` | 전체 확정 | 1 클럭 |
| `in_port[6]` | 취소 | 1 클럭 |
| `in_port[7]` | 비밀번호 변경 | 1 클럭 |
| `in_port[8]` | 마스터키 | 1 클럭 |

> 버튼 신호가 1클럭만 유지되므로, CPU assembly에서 `IN 1` 같은 폴링 루프를 돌면 latch가 SET된 바로 그 사이클에만 1이 읽힌다. 그 이후 사이클에는 0.

## 3. top_cpu (CPU 코어)

**역할**: assembly 프로그램을 실행하고, 도어락 상태를 `out_port`로 출력

### 서브모듈 구성

```
top_cpu
├── cpu_fsm       : FETCH/DECODE/EXECUTE/INCREMENT 상태 제어
├── pc            : Program Counter
├── inst_reg      : Instruction Register (IR)
├── decoder       : opcode 해석 → 제어 신호 생성
├── accumulator   : ACC 레지스터
└── alu           : 산술/논리 연산
```

### CPU FSM 상태 전이

```
FETCH(00) → DECODE(01) → EXECUTE(10) → INCREMENT(11) → FETCH(00) → ...
```

매 상태 1클럭. `clk_enable = 1`이면 항상 진행.

---

## 4. CPU 파이프라인 단계별 동작

### FETCH (state = 00)

```
bram_addr ← pc_out        (현재 PC를 BRAM 주소로 설정)
bram_we   ← 0
```

BRAM은 동기식 read이므로, 이 사이클에 주소를 넣으면 다음 사이클(DECODE)에 데이터가 출력된다.

### DECODE (state = 01)

```
ir_we ← 1
IR ← bram_rdata            (BRAM에서 읽어온 32비트 명령어를 IR에 저장)
```

decoder는 `inst_reg`의 출력(`instr_out`)을 조합논리로 항상 디코딩하고 있으므로, IR이 업데이트된 직후(EXECUTE 사이클)부터 제어 신호가 유효해진다.

```
decoder 출력 (조합논리, 항상 계산):
  opcode = instr_out[31:28]
  imm    = instr_out[27:0]
  addr   = instr_out[11:0]
  port   = instr_out[3:0]
  → is_load, is_store, is_alu_mem, is_alu_imm, is_jump, is_in, is_out
  → alu_ctrl, zero_we, jump_type
```

### EXECUTE (state = 10)

명령어 타입에 따라 BRAM 주소와 write enable을 설정한다.

| 명령어 타입 | bram_addr | bram_we |
|---|---|---|
| LOAD / ALU_MEM | `addr` (operand[11:0]) | 0 |
| STORE | `addr` | 1 |
| 그 외 (IMM, JMP, IN, OUT, NOP) | `pc_out` | 0 |

STORE 실행 시: `bram_wdata = acc_out` (ACC 현재값이 BRAM에 기록됨)

ALU는 조합논리로 항상 동작:

```
alu_operand ← is_alu_imm ? {4'b0000, imm}   (ADDI/CMPI/LOADI: 28비트 즉시값)
            : is_ext_imm ? {20'b0, addr}     (SHL/SHR: 12비트 시프트량)
            :               bram_rdata        (ADD/SUB/CMP/AND: 메모리값)
alu 입력: acc_out, alu_operand, alu_ctrl
alu 출력: alu_result, alu_zero_result  (combinational, 항상 유효)
```

### INCREMENT (state = 11)

이 사이클에 레지스터 쓰기가 집중된다.

#### ACC 업데이트

| 명령어 | acc_in | acc_we |
|---|---|---|
| LOAD | `bram_rdata` (EXECUTE에서 addr→BRAM, 이 사이클에 데이터 출력) | 1 |
| IN | `selected_input` (in_port 선택값) | 1 |
| ADD / SUB / ADDI / LOADI | `alu_result` | 1 |
| SHL / SHR / AND | `alu_result` | 1 |
| CMP / CMPI | — | 0 (ACC 보존) |
| 그 외 | — | 0 |

`IN` 포트 선택 로직:

```
port = instr_out[3:0]

IN 0 → selected_input = {28'd0, in_port[3:0]}   ← 숫자값
IN 1 → selected_input = {31'd0, in_port[4]}     ← 자리 입력 버튼
IN 2 → selected_input = {31'd0, in_port[5]}     ← 전체 확정 버튼
IN 3 → selected_input = {31'd0, in_port[6]}     ← 취소 버튼
IN 4 → selected_input = {31'd0, in_port[7]}     ← 비밀번호 변경
IN 5 → selected_input = {31'd0, in_port[8]}     ← 마스터키
```

#### ZERO_FLAG 업데이트

`zero_we = 1`인 명령어(ADD, SUB, CMP, ADDI, CMPI, SHL, SHR, AND)에서만 업데이트:

```
zero_flag ← alu_zero_result
```

#### out_port 업데이트

`is_out = 1` (OUT 명령어)일 때:

```
out_port ← acc_out[3:0]
```

#### PC 업데이트

```
기본: pc_next = pc_out + 1

is_jump = 1이면:
  JMP (UNCOND) : pc_next = addr
  JZ           : zero_flag ? addr : pc_out + 1
  JNZ          : !zero_flag ? addr : pc_out + 1

pc_out ← pc_next  (pc_write = 1)
```

## 5. 명령어 타입별 파이프라인 요약

### LOAD addr

```
FETCH     : bram_addr = PC         → BRAM에서 명령어 읽기 시작
DECODE    : IR ← bram_rdata        → IR = LOAD 명령어
EXECUTE   : bram_addr = addr       → BRAM에서 데이터 읽기 시작
INCREMENT : ACC ← bram_rdata       → 데이터 ACC에 저장, PC++
```

### STORE addr

```
FETCH     : bram_addr = PC
DECODE    : IR ← bram_rdata
EXECUTE   : bram_addr = addr, bram_we = 1, bram_wdata = ACC  → BRAM에 ACC 기록
INCREMENT : PC++
```

### ADD / SUB / CMP addr  (ALU_MEM)

```
FETCH     : bram_addr = PC
DECODE    : IR ← bram_rdata
EXECUTE   : bram_addr = addr       → BRAM에서 피연산자 읽기 시작
INCREMENT : bram_rdata → ALU 연산 → ACC (CMP는 생략), ZERO_FLAG, PC++
```

> EXECUTE에서 bram_addr=addr로 설정하면, BRAM은 다음 사이클(INCREMENT)에 데이터를 출력한다.
> ALU는 조합논리이므로 INCREMENT 시작 시점에 bram_rdata가 확정되는 즉시 alu_result가 유효해진다.

### LOADI / ADDI / CMPI  (ALU_IMM)

```
FETCH     : bram_addr = PC
DECODE    : IR ← bram_rdata
EXECUTE   : alu_operand = {4'b0, imm}  (BRAM 접근 없음)
INCREMENT : ALU 연산 → ACC (CMPI는 생략), ZERO_FLAG (ADDI/CMPI만), PC++
```

### JMP / JZ / JNZ

```
FETCH     : bram_addr = PC
DECODE    : IR ← bram_rdata
EXECUTE   : (특별한 동작 없음)
INCREMENT : jump_type, zero_flag에 따라 pc_next 결정 → PC ← pc_next
```

### IN port

```
FETCH     : bram_addr = PC
DECODE    : IR ← bram_rdata
EXECUTE   : (특별한 동작 없음)
INCREMENT : selected_input = in_port[port 번호에 따라 선택] → ACC ← selected_input, PC++
```

### OUT port

```
FETCH     : bram_addr = PC
DECODE    : IR ← bram_rdata
EXECUTE   : (특별한 동작 없음)
INCREMENT : out_port ← ACC[3:0], PC++
```

### SHL / SHR amount  (EXT_IMM)

```
FETCH     : bram_addr = PC
DECODE    : IR ← bram_rdata        → opcode=1111, funct=0000(SHL)/0001(SHR)
EXECUTE   : alu_operand = {20'b0, addr}  (시프트량, BRAM 접근 없음)
INCREMENT : ALU 연산 → ACC ← alu_result, ZERO_FLAG, PC++
```

### AND addr  (EXT_MEM)

```
FETCH     : bram_addr = PC
DECODE    : IR ← bram_rdata        → opcode=1111, funct=0010
EXECUTE   : bram_addr = addr       → BRAM에서 피연산자 읽기 시작
INCREMENT : bram_rdata → ALU AND 연산 → ACC ← alu_result, ZERO_FLAG, PC++
```

## 6. output_handler

**역할**: CPU `out_port[3:0]` 상태 코드를 물리 출력(도어, LED)으로 변환

### 입력

| 신호 | 폭 | 설명 |
|---|---|---|
| `out_port` | 4 | CPU가 OUT 명령어로 기록한 상태 코드 |

### 처리 (동기식 case 디코딩)

```
posedge clk:
  case(out_port)
    0000 → door_open=0, led=0000  ← 닫힘
    1000 → door_open=1, led=1000  ← 열림 (LD3)
    0100 → door_open=0, led=0100  ← 1회 오답 (LD2)
    0010 → door_open=0, led=0010  ← 2회 오답 (LD1)
    0001 → door_open=0, led=0001  ← 3회 오답 + 잠금 (LD0)
```

### 출력

| 신호 | 폭 | 핀 | 설명 |
|---|---|---|---|
| `door_open` | 1 | JD10 V18 | 외부 도어락 제어 |
| `led[3:0]` | 4 | LD3~LD0 | 온보드 상태 표시 LED |

## 7. BRAM 타이밍 정리

BRAM은 동기식 single-port read:

```
클럭 엣지 N   : bram_addr 설정
클럭 엣지 N+1 : bram_rdata 출력 유효
```

CPU 파이프라인 대응:

| 용도 | 주소 설정 사이클 | 데이터 유효 사이클 |
|---|---|---|
| 명령어 fetch | FETCH | DECODE (→ IR 래치) |
| 메모리 read  | EXECUTE | INCREMENT (→ ACC 래치) |
| 메모리 write | EXECUTE (bram_we=1) | BRAM 기록 완료 |

## 8. 전체 신호 흐름 요약

```
물리 입력
  └─ input_handler
       ├─ debouncer × 15개 (pmod 10 + 버튼 5)
       ├─ casez 우선순위 인코더 → key_num[3:0]
       └─ self-clearing latch × 5 → latch_*
                │
                ▼
           in_port[8:0]
                │
                ▼
           top_cpu
             ├─ cpu_fsm    : 4-state 순환 (1클럭/state)
             ├─ pc         : FETCH에서 bram_addr 공급, INCREMENT에서 갱신
             ├─ inst_reg   : DECODE에서 bram_rdata 래치
             ├─ decoder    : IR → 제어 신호 (조합논리)
             ├─ alu        : ACC op operand → result, zero_result (조합논리)
             └─ accumulator: INCREMENT에서 갱신
                │            │
                │            ▼
                │       bram_addr / bram_wdata / bram_we
                │            │
                │            ▼
                │          BRAM (doorlock.coe, 4096×32bit)
                │            │
                │       bram_rdata
                │
             out_port[3:0]
                │
                ▼
          output_handler
             └─ case 디코딩 → door_open, led[3:0]
```
